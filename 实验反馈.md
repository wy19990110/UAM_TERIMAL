According to a document from 2026-04-03，这篇稿子的目标已经定死了：研究对象是**低空可复用航路网络设计中的非点终端问题**，核心是回答“把终端当普通节点会不会误导航路网络设计”“A/S/F 哪些通道会穿透到上层”“什么接口层级足以保持网络决策基本正确”。因此，新的实验设计必须服务于 **decision-preserving interface**，而不是再做一次“快但辨识不清”的全扫。实验链原本也明确是：EXP-0 做 factor screening，EXP-1/2/3 做 A/S/F 的存在性机制，EXP-4 做 regime map，EXP-5 做现实代理案例。

我先给结论：**现在不是在现有 EXP-4 上补几个 patch，而是要把 EXP-4 重构成“数值校准 + 通道隔离 + 混合 regime + 现实代理”四段式包。** 这样做不是为了硬保三段式，而是为了把“AS 是否真的形成独立区间”这件事做成**可识别**的问题；如果最后还是两段式，那也是可信结论，而不是被实现偏差和图生成器带出来的假象。

先说为什么当前 EXP-4 不能直接当 TRC 主证据。现有版本用 `nPwl=7`、`intlinprog + quadprog`，972 个实例约 2.5 分钟跑完；结果是 M0 60%、M1 59%、M2 91%，而且 αA、κS 几乎不分离，真正明显的只有 φF。报告自己也已经承认：M1 并未明显优于 M0，αA/κS 几乎无效，ρ=0.5 还出现反常；结论呈现成了“两段式”，而不是原本想要的三段式 regime map。

问题清单把根因说得很清楚：一是 PwL 近似本身带来约 10% 的 baseline regret；二是 `buildSynthetic` 把 port 方向对齐 incident edge，导致 admissibility 对 M0 几乎不起约束；三是 κS 的效应被当前线性化稀释；四是单商品流可能进一步冲淡 regret 差异。也就是说，当前 EXP-4 的主要问题不是“结果不好看”，而是**识别和数值都不够干净**。

但同时，EXP-1/2/3 的存在性证据是有效的，不能扔。EXP-1 已经证明 A 缺失会导致 infeasible 或巨大 regret；EXP-2 已经在精确 MIQP 口径下证明，A 相同时，aggregate service 会系统性误导，M1 在 64 个 M0 有正 regret 的实例里 recovery rate 是 100%；EXP-3 已证明 footprint 在 hard-block/soft penalty 下都可能改变拓扑，而且 M2 能恢复、M1 不能。这些东西是新设计必须保留的“机制地基”。

下面是我建议的**重构版实验方案**。

## 一、先把论文 claim 改成“模型升级规则”，不要把主命题绑死在假 J* 上

TRC 更容易接受的主问题不是“我证明了一个全局最小充分接口”，而是：**在什么类型的问题上，M0 够；什么时候值得升级到 M1；什么时候必须 M2。** 这和你们交接文档里的主线完全一致：关键是哪个 channel 最关键、regret 在什么 regime 下显著、哪一层接口足够恢复 benchmark 的大部分设计价值。

所以主指标不要再把分析压在当前那个近似口径下的 (J^*) 上。正式主指标改成两个 pairwise uplift：

[
U_{01}=J^{truth}(M0)-J^{truth}(M1),
\qquad
U_{12}=J^{truth}(M1)-J^{truth}(M2)
]

然后用它直接定义推荐模型：

* 若 (U_{01}<3%) 且 (U_{02}<3%)，推荐 M0
* 若 (U_{01}\ge 3%) 且 (U_{12}<3%)，推荐 M1
* 若 (U_{12}\ge 3%)，推荐 M2

这样你回答的是“值不值得升级模型”，而不是拿一个目前并不干净的 benchmark 硬压全部结论。相对 regret 对 exact truth benchmark 仍然保留，但只放在**校准子集**和附录里做 secondary evidence。这个改法能直接规避 ISSUE-4 那种“best among compared models 伪装成 J*”的问题，同时更贴近论文真正要交付的 regime 规则。

## 二、把 EXP-4 拆成四个子实验，不再做 blind sweep

### EXP-4A：数值校准与 benchmark 校验

这一步是新的入口，不通过就**不准**解释 regime。

目标只有一个：把当前 `PwL vs truth QP` 造成的 baseline regret 压下去。问题清单已经明确指出，φF=0 时 M0/M1/M2 理应等价，但现在仍只有约 90% sufficiency，这就是共享的数值地板，不是科学信号。

我建议这样做：

1. 统一求解栈。EXP-4 主线改到 **Python + Gurobi MIQP/MIQCP**。既然 EXP-2/3 已经在 Gurobi 精确二次目标下跑通，EXP-4 就不要再用 MATLAB `intlinprog + 7 段 PwL` 当主证据了。近似只能当加速器，不能当主分析口径。

2. 建一个 **exact calibration subset**。从 3 个图族里抽 40–60 个代表性实例，覆盖 φF=0、A 高、S 高、F 高四类情形。对这批实例用 exact/near-exact MIQCP 求解 M0/M1/M2/M*。

3. 如果全量仍需近似，则只允许 **SOS2/convex-combination** 或更高精度凸近似，不再用简单切线下界当正式口径；同时必须做 `nPwl=7/15/31` 的收敛曲线。

4. 这一步的过关标准我建议写死：

   * φF=0 子集上，M0/M1/M2 的中位 baseline regret < 1%
   * 95 分位 < 3%
   * 精确子集上不再系统性出现 “M1 > M0”
     只要没过，后面任何 regime 图都不解释。

这一段本质上是把现在的“~10% baseline regret”从 discussion 里的借口，变成实验前置单元测试。

### EXP-4B：A/S 隔离的 medium regime map

当前 EXP-4 最大的问题是把 A/S/F 全混在一起扫，最后只有 F 被激发出来。任务书原本确实是想在完整网络上回答 S-only / AS / ASF 的适用区间，但现在这件事必须先拆开做，先确认 A 和 S 到底有没有被 medium family 真正激发。

这一步我建议再拆成两个 family。

**B1：A-only family**

设定：

* 强制 φF=0
* service 差异压小，让 S 不主导
* port 不再对齐 incident edge
* 每个 port 增加 1–2 条 non-incident connector，让 M0 真正拥有“错误选项”

新的 generator 不要再追求“看起来随机”，而要追求“admissibility 有辨识度”。问题清单已经指出，当前图生成器最大的毛病就是 port 对齐 incident edge，导致 M0 和 M1 的 connector 选择集合几乎一样。

A-family 里，除了 αA 本身，我建议加一个**实际激发指标**：

[
E_A = 1 - \frac{|C_{M0}\cap C_{M1}|}{|C_{M0}\cup C_{M1}|}
]

也就是 M0 可见 connector 集和 M1 admissible connector 集的 Jaccard 距离。没有这个指标，就根本不知道某个实例有没有资格拿来说明 A。主图不再是单纯 αA 横轴，而是 `ρ × E_A` 或 `αA × E_A` 的 uplift heatmap。

**B2：S-only family**

设定：

* 强制 φF=0
* 所有 connector 全 admissible
* 保留精确的 port-level 二次 service、cross-port coupling、饱和阈值
* 用 exact MIQP，不允许 terminal service 在这一步被粗线性化

EXP-2 已经告诉你：S 的存在性问题是真的，而且 ρ 从 0.4 到 0.6 有清晰跳变，M1 recovery rate 100%。所以 B2 的任务不是“再证明一次 S 会不会重要”，而是把这种机制扩展到 medium 图里，并确认当前 EXP-4 为什么没有看见它。

但我建议这里**不要再把 κS 当唯一横轴**。因为 EXP-2 已经显示，κS 不是单调意义上的“越大越糟”，最大 regret 反而可能出现在对称/可分流区间。更好的做法是定义一个 **service excitation score**，比如 nominal load 下各 port marginal cost 的离散度 + cross-port coupling 强度，记成 (E_S)。最终画 `ρ × E_S` 上的 (U_{01}) 热图，回答“什么时候从 M0 升到 M1 值得”。

B 家族的过关标准很简单：**只要 medium 条件下在一大片高 (E_A)/高 (E_S) 区域里，M1 相比 M0 有稳定正 uplift，就说明 AS 中间层是可识别的。** 若修完后仍分离不出来，那时再说 AS 不形成独立区间，才有学术含义。

### EXP-4C：F 隔离的 medium regime map

这一块其实是你们现在最强的，所以不要推倒重来，而是做两件升级。

第一，延续 EXP-3 的机制：A 和 S 固定，只让 footprint 变。EXP-3 已经明确给出了 F 的两种触发机制：hard-block 和 soft penalty，而且在 φF=1.0 hard-block 下 M1 catastrophic failure，M2 恢复。这是 paper 的硬证据之一。

第二，把 M2 的 F 接口做得再干净一点。当前 EXP-3 已经自己写了，M2 在 soft footprint 下还有 5.9% 残差，不是 bug，而是因为 M2 只提 nominal penalty，没有提 load-sensitivity。既然第一篇的主线已经固定围绕 ASF，这个 F 维度就没必要故意做弱。我的建议是把 M2 的 footprint 接口从

[
F=(\tilde\pi,\tilde B)
]

升级成

[
F=(\tilde\pi,\tilde B,\tilde\rho)
]

也就是多提一个 edge-level load-sensitive coefficient。它仍然是 F，不会破坏论文接口定义，但会显著减少“明明是主张 ASF，却让 M2 因为接口提取太弱而吃亏”的假阴性。

C 家族的主图就是 (U_{12}) 的 heatmap，横轴用 φF 或 (E_F)，纵轴用 ρ。再加 blocked-edge violation rate 和 topology distance。这里你们本来就已经有基础，重做成本最低。

### EXP-4D：混合 regime map

只有 4A、4B、4C 都过关以后，才允许做混合全网图。
而且这一步**不要再做 4×3×3×3 的 blind full factorial**。任务书原本是 1620 个实例，但现在更重要的是“可解释”而不是“多”。

我建议把 mixed regime 重构成 **4 个图族 × 4 个 seed × 24–36 个设计好的组合**，总量 384–576 个实例，宁可少一点，也要可信。

图族建议改成：

* G1：open sparse generic
* G2：open dense generic
* G3：access-sensitive urban（故意保留 off-axis ports 和 extra connectors）
* G4：airport-adj footprint-sensitive（近机场保护区与 transition constraints）

每个实例除了原始参数，还要计算三个**实际激发指标**：

[
E_A,\quad E_S,\quad E_F
]

最后 mixed regime 不是在 `αA, κS, φF` 的名义参数空间里解释，而是在 `E_A, E_S, E_F` 的实际激发空间里解释。这样图才能读，故事才能讲，审稿人才会相信你看到的真的是 channel effect，而不是 generator artifact。

最后一张总图，不是 “哪个模型 regret 最小”，而是 **model recommendation map**：每个 excitation 区域推荐 M0/M1/M2 哪一个。这样和论文主问题完全对齐。

## 三、EXP-0 必须补上，而且要放回主线

这一步不是可有可无。交接文档已经写得很明白：没有 factor–channel 这一层，ASF 就会像作者凭感觉挑的三个字母；正文必须有 factor–channel matrix 和从 (u_t) 到 (K_t) 的代数定义。实验任务书也已经给出了 EXP-0 的具体设计：600 个 base terminals，逐个做 one-family perturbation，输出 (d_A,d_S,d_F) 并做 effect size 排名。

所以我的建议是：**EXP-0 不再被视作“有空再补”，而是跟 4A 一起变成主线前置条件。**
没有它，这篇论文在 TRC 那边的方法论根基不够硬。

## 四、EXP-5 变成必做，不再可选

实验安排和任务书都已经写得很直接：现实代理案例“一定要有”，目的是给编辑一个“这不是纯 synthetic 机制体操”的场景，而且环境都已经想好了：1 个机场保护区，2 个 procedure-like terminals，2–3 个 path-based vertiports，low/medium/high × relaxed/constrained 共 6 个情景；输出是 overlay、cost breakdown、truth-evaluated regret，以及哪些边被 A 和 F 改写。

所以这里我建议得更明确一点：
**EXP-5 必须进主文，不要再把它当成“最后有空做”。**

原因很简单：TRC 审稿人不满足于 “synthetic 上存在机制”，他们会追问“那实际网络图到底怎么变”。EXP-5 正好回答这个问题，而且还能把 paper 从“抽象 interface 论文”推进到“有交通系统解释力的 network design 论文”。

## 五、把单商品流从正式版 EXP-4 里拿掉

问题清单把单商品流列成 P2，但如果目标是拿去投 TRC，我会把它升到 P1.5。因为你们的论文对象是**网络设计**，OD conservation 如果被 aggregate 掉，审稿人很容易质疑 topology 的解释力。问题清单自己也承认，单商品流允许不同 OD“搭便车”，会压低所有模型的目标值，从而缩小 regret 差异。

我的建议是：

* debug 阶段可以继续单商品
* 但最终主文版 EXP-4D 和 EXP-5，必须上 **multi-commodity flow**
* 如果算不动，就减少实例数或图规模，不要为了跑得快牺牲 OD 真实性

TRC 更看重“可信的网络结论”，不太看重“972 个实例 2.5 分钟”。

## 六、EXP-1/2/3 不需要推翻，但要统一口径和写法

我不建议重写这三个实验的逻辑，因为它们现在的作用已经很清楚：
EXP-1 是 A 的硬机制；EXP-2 是 S 的渐变机制；EXP-3 是 F 的拓扑机制。它们正好构成 paper 的三段命题证明。只是为了主文整洁，建议在最终版里统一：

* 全部在同一套 Python/Gurobi 代码栈里重跑一个 clean version
* 统一输出 truth-evaluated objective、pairwise uplift、topology distance
* 在正文里把它们作为 **mechanism propositions**，而不是和 EXP-4 混成同一种统计证据

这样 EXP-4 就专心回答 regime，EXP-1/2/3 专心回答 existence。

## 七、最终可以支撑 TRC 的论文包长什么样

如果按上面重做，主文结构我建议就是 7 张主图：

1. factor → channel → interface 框架图 + factor–channel matrix
2. EXP-1/2/3 的三个最小反例
3. 数值校准图：exact vs approximate / baseline convergence
4. A/S 隔离家族上的 (U_{01}) regime map
5. F 隔离家族上的 (U_{12}) regime map
6. Mixed family 的 model recommendation map
7. EXP-5 现实代理 overlay

附录再放：

* approximation convergence
* exact subset details
* ASF 对抗性证伪（原计划的 EXP-6）
* solver/optimality gap 说明

这样整篇稿子的证据链会非常完整：
**为什么是 ASF → A/S/F 各自为什么重要 → medium 里什么时候该升级模型 → 现实上网络到底怎么变。**

## 八、执行顺序我建议这样排

第一步，补 EXP-0。
第二步，做 EXP-4A 数值校准。
第三步，做 EXP-4B A/S 隔离家族。
第四步，做 EXP-4C F 隔离家族。
第五步，做 EXP-4D mixed regime。
第六步，做 EXP-5 现实代理。
第七步，有余力再做 EXP-6 附录。

一句话压缩：**不要再拿当前这个 blind medium sweep 去“讲论文”；把 EXP-4 重构成一个有校准、有激发、有隔离、有现实代理的实验包。**
这套设计不是为了保证你一定得到三段式，而是为了保证：**无论最后是三段式还是两段式，结论都能在 TRC 里站得住。**

如果你要，我下一条可以直接把这个方案整理成一个“新版实验任务书”，按“输入—生成器—求解器—指标—过关标准—图表输出”六栏给你。
