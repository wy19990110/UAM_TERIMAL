## 摘要的四句话结构

### 第一句：背景

低空/UAM 正从点状示范走向网络化建设，但现有网络设计通常把 terminal（终端）视为同质节点，而现实终端的接入、服务和邻域约束明显异质。

### 第二句：问题

因此，关键问题不是把 terminal 整体抬升进网络优化，而是找出：**为了把网络设计做对，终端最少需要向上层暴露什么信息。** 

### 第三句：方法

本文提出并检验一个分层终端接口：A（可接性）、S（端口级服务）、F（邻域脚印/冲突），统一比较 S-only、AS、ASF 与 truth benchmark，采用“固定设计、统一 truth 重算”的评估协议。

### 第四句：结论

最小反例表明三条通道均可独立误导网络设计；中型与全网实验表明 A 和 F 的模型升级边界可清晰识别，现实代理案例进一步显示机场邻近约束会实质改写网络拓扑。EXP-0 只作为 adopted interface hypothesis 的筛查支持，而非发现性结果。

## 1. 引言（Introduction）

### 1.1 背景

低空交通与 UAM 正从 isolated demonstrations（孤立示范）进入 network design（网络设计）阶段，城市需要考虑 reusable routes / corridors（可复用航路/走廊）、terminal siting（终端布局）与接入组织的协同。

### 1.2 文献断层

现有研究要么把 terminal interface（终端接口）当固定边界来设计外部网络，要么研究 terminal capacity / operations（终端容量/运行），但二者之间缺少一个供网络设计直接调用的中层接口。

### 1.3 核心矛盾

网络设计希望 terminal 越粗越好，但 terminal 异质性又会穿透到网络层。问题于是变成：
**终端需要向网络暴露什么，才足以保持设计基本正确？** 

### 1.4 本文贡献

我建议写成四条：

1. 提出一个面向网络设计的分层终端接口 A/S/F；
2. 提出一个 design-evaluate-under-truth（在真值下评估设计）的统一评估协议；
3. 给出从存在性、通道隔离到全网分区的完整证据链；
4. 形成“何时用 M0/M1/M2”的模型升级规则。

### 1.5 边界声明

本文不解决 terminal layout / scheduler 的最优设计，也不声称 A/S/F 是全局最小充分接口；本文研究的是 adopted truth class 下的 interface sufficiency for strategic network design（战略网络设计中的接口充分性）。

## 2. 相关研究（Literature and positioning）

### 2.1 低空/UAM 网络设计

关注 reusable routes / corridor planning / network design。
定位你们的问题属于“network design under heterogeneous terminal interfaces（异质终端接口下的网络设计）”。

### 2.2 terminal capacity / operations abstraction

说明终端区能被压成服务/延误对象，但现有文献通常不把这种压缩直接连接到网络拓扑设计。

### 2.3 本文位置

你们不做：

* 新调度算法
* 新微观仿真器
* 新候选图生成算法

你们做的是：

* **一个网络可用的终端接口层级**
* **一个面向设计误差的验证框架**

TRC 的定位强调 transportation systems 的 design / planning implications（设计/规划含义），这正是这篇最应该强调的点。([科学直接][1])

## 3. 问题定义与接口思想（Problem statement and interface philosophy）

### 3.1 上层对象

完整候选网络上的 strategic route design（战略航路设计）：

* 选 backbone（主干边）
* 选 connectors（接入边）
* 分配流量

### 3.2 为什么 terminal 不能再是点

这里用一张概念图，画：

* 可接性差异
* 端口级服务差异
* 邻域约束差异
  说明它们都会改变上层设计排序。

### 3.3 A/S/F 的先验来源

这里一定要写清：

> ASF 是 theory- and design-motivated interface hypothesis（理论和设计需求驱动的接口假设），不是数据驱动发现。

### 3.4 四个模型

* M0：S-only
* M1：AS
* M2：ASF
* M*：truth benchmark

---

## 4. 采用的终端真值类与接口提取（Adopted terminal truth class and interface extraction）

### 4.1 这节的目标

不是证明终端运行的终极机理，而是定义一个：

* 结构上比 M0/M1/M2 更丰富，
* 对网络设计足够敏感，
* 但仍可优化的中观真值类。

### 4.2 A：接入可行性

写成几何+规则的 admissibility object（可接性对象）。

### 4.3 S：服务能力

这节只做两层写法：

* 资源层：共享资源与 port-resource incidence（端口—资源占用）
* 降阶层：port-level convex cost + cross-port coupling（端口级凸成本+耦合）

### 4.4 F：脚印/冲突

主文收窄成：

* hard blocks（硬阻断）
* nominal local penalties（名义局部惩罚）
  load-sensitive footprint 放附录或扩展。

### 4.5 M0/M1/M2 怎么从 truth 提取

把 3 层接口清楚写出来。

这一节就是“方法的基石”。但注意：
**它是 adopted truth class（采用的真值类），不是现实终端运行的唯一正确模型。**

---

## 5. 评估协议（Evaluation protocol）

### 5.1 共用候选图

所有模型共享同一 candidate graph（候选图）。

### 5.2 先设计、后真值重算

强调：

* 不是比模型自己的目标值
* 而是比在同一 truth 下固定设计的表现

### 5.3 核心指标

* truth-evaluated objective
* regret
* recovery rate
* topology distance
* sufficiency probability（充分率）

### 5.4 为什么这比“拟合误差”更对题

因为你研究的是**网络有没有被误导**。

## 6. EXP-0：接口筛查（Screening only）

### 6.1 角色

必须降格成：

* adopted interface hypothesis 的 screening / sanity check
  而不是：
* 发现 ASF
* 证明 factor→channel

### 6.2 当前做法

600 个 base terminals，5 个粗 family，单 terminal、无网络求解、输出 `d_A/d_S/d_F`。

### 6.3 正文怎么写

只放：

* 一张 family × channel 响应图
* 一段定位说明
  更多放附录。

### 6.4 该节边界

明确写：
它只说明 ASF 不是空接口，不说明 factor→channel 已被严密证明。

## 7. EXP-1/2/3：三条通道的存在性命题

### 7.1 EXP-1：A 的必要性

用 hard-cut / soft minimal counterexamples（硬切/软最小反例）证明 omission of admissibility 会直接改错 connector / topology。

### 7.2 EXP-2：S 的必要性

在 A 固定、图固定下，只因为 port-level service 不对称，M0 就会在中高负荷下系统性犯错。

### 7.3 EXP-3：F 的必要性

在 A、S 相同下，local footprint 足以把 near-edge 方案推成 bypass 方案，M1 失效、M2 恢复。

这三节最好写成：

* proposition-style numerical evidence（命题式数值证据）

## 8. EXP-4：从隔离机制到模型升级分区

### 8.1 4A：校准门控

证明 baseline 已足够低，后面分区图不是近似误差。当前 4A 已通过：在 `φ_F=0` 子集上，M1/M2 的 median 和 P95 regret 都接近 0。

### 8.2 4B：A 的中型边界

当前主要打出了 A，不是 S。关键结论是：`E_A≈0.7` 左右，M0→M1 的升级值开始显著。

### 8.3 4C：F 的中型边界

`φ_F≈0.5` 左右，M1→M2 的升级值显著；但中等 `φ_F` 有 McCormick 伪影。

### 8.4 4D：混合适用区间图

## 9. EXP-5：现实代理案例

### 9.1 角色

现实解释力，不是 general proof（普适证明）。

### 9.2 当前 strongest case

`high_constrained` 强力支撑 F-heavy airport-adjacent proxy（F 主导的机场邻近代理场景）：M2 远优于 M0/M1。

### 9.3 要老实写的限制

当前单一 proxy 里 `M0≈M1`，所以它主要支撑 F，不支撑 A/S 的现实一般规律。

## 10. 讨论（Discussion）

这节必须有，而且要干四件事：

### 10.1 这篇证明了什么

* terminal 不是点
* 三条通道各自都可能误导网络
* A 和 F 的升级边界已在 adopted class 中识别
* 现实代理显示 footprint-aware design（脚印感知设计）会改网

### 10.2 这篇没有证明什么

* 没有证明 ASF 是全局最小充分接口
* 没有证明 S 的独立 medium-scale 边界已经锁死
* 没有证明 truth class 是现实世界唯一正确机理

### 10.3 EXP-0 的真实地位

它是 adopted interface hypothesis 的 screening，不是 discovery module（发现模块）。

### 10.4 F 的方法边界

主文 F 是“几何/规则驱动的局部外部性类”，不是完整 terminal-side conflict science（终端冲突科学）。

---

## 11. 结论（Conclusion）

最稳的结尾不是“我们证明了 ASF 必要”，而是：

> 对低空可复用航路网络设计而言，异质终端不能再被视为点节点；
> 一个分层终端接口能够把终端复杂性压缩成对网络真正有用的设计信息；
> 在采用的真值类中，A 和 F 通道分别对应明确的模型升级边界，而终端区内部复杂性不必整体抬升到上层网络优化。
