# 论文完整说明：背景、目的、想定、算法、结果

## 一、论文背景

### 1.1 领域问题

城市空中交通（UAM / Advanced Air Mobility）需要设计**可复用的低空航路网络**——一组固定走廊（corridors），连接分布在城市中的垂直起降场（vertiports）。

现有文献分成三个互不说话的阵营：
- **外部航路网络设计**（He 2024/2025 等）：把终端当成简单节点，给个容量上限就完事
- **终端区容量与调度**（Vascik/NASA 等）：关注起降坪/停机位的时间线调度，但不管外部网络怎么接
- **终端区运行构型**（Qu、Shao 等）：研究等待环、进离场定位点、多环结构，但没有压缩成上层网络设计可调用的接口

**断层在于**：终端区不是一个简单的"容量节点"。终端的构型（approach layout、holding pattern）会决定哪些走廊能接入、空域脚印会不会挡住邻近走廊、不同进近口的延迟差异有多大。但如果在上层网络设计中塞入完整终端微观仿真，又太重、解不动。

### 1.2 核心研究问题

> **为了设计低空可复用航路网络，终端区到底需要被抽象到什么粒度？一旦这个最小充分粒度被找到，工程上是否可以重新回到分阶段设计（先设计终端，再设计网络）？**

### 1.3 论文不做什么

- 不做全城逐机 4D 微观仿真
- 不做 PDE/NSE 级气动分析
- 不做程序批准链（8168/TERPS）
- 不是在造一个更大的联合规划器

### 1.4 论文的交付物

1. 一套**终端区中层抽象层级体系**（A0 → A1 → A2 → A2+）
2. 一套**"抽象误差如何传到网络设计决策误差"的理论框架**（基于 regret / 决策保持）
3. 一个核心结论：**何时只看容量就够，何时必须把接口、空域脚印、外部性带进来**
4. 一个工程建议：**一旦最小充分接口成立，分阶段设计重新成立**

### 1.5 理论脊梁

"决策保持抽象"不是一个现成领域，而是四条成熟理论线的交叉：

1. **可聚合性 / 状态聚合**（lumpability, MDP model minimization）：什么条件下聚合是精确的
2. **近似双模拟**（approximate bisimulation）：抽象不必复制细节，只要可观察行为误差有界
3. **优化稳定性**（Lipschitz stability）：参数扰动（终端抽象误差）→ 最优解扰动（网络设计偏差）
4. **决策误差 / predict-then-optimize**：关心的不是预测误差本身，而是它诱导的 regret

核心数学关系：若 sup|J(y;φ) - J(y;φ̂)| ≤ ε_J，则 regret Δ ≤ 2ε_J。若最优解与次优解间隔 g > 2ε_J，则最优设计本身不变。

## 二、抽象层级定义

| 层级 | 符号 | 可见信息 | 直觉含义 |
|------|------|----------|----------|
| **A0** | φ₀=(μ,D) | 总容量 + 聚合延迟 | 终端是黑箱，只告诉你"能处理多少架、平均等多久" |
| **A1** | φ₁=(A,μ,D) | + 接口可行矩阵 + 接口级延迟 | 知道哪条走廊能接哪个进近口，各口延迟不同 |
| **A2** | φ₂=(A,μ,D,V,X) | + 空域脚印 + 外部性 | 知道终端场型的空域占用会阻塞附近走廊、噪声影响 |
| **A2+** | φ₂⁺=(A,μ,D,V,X,C) | + 资质/关闭/应急规则 | 异构终端混合运行（直升机场 + UAM 终端等） |
| **Full** | 完全信息 | 一切可见 | 真值基准，用于评估其他层级的决策偏差 |

**层级之间是信息包含关系**：A0 ⊂ A1 ⊂ A2 ⊂ A2+ ⊂ Full

## 三、要验证的结论

### 主张 P1（A0 不充分存在性）
存在网络实例，使得仅用 A0 信息求解的最优走廊设计，在真值评估下产生正 regret。
→ 仅看容量不够，接口信息有时是必须的。

### 主张 P2（A1 不充分存在性）
存在网络实例，使得仅用 A1 信息求解的最优走廊设计，在真值评估下产生正 regret。
→ 接口信息也不够，空域脚印有时是必须的。

### 主张 P3（Regime Map）
在不同的（需求强度 × 脚印约束水平）组合下，**最小充分抽象层级不同**。
→ 不是"A2 总是必须"或"A0 总是够"，而是取决于运行条件。

### Regret 的数学定义

```
Δ_ℓ = J(ŷ_ℓ; φ_Full) - J(y*; φ_Full)
```

- y* = argmin J(y; φ_Full)：Full 模型下的全局最优网络设计
- ŷ_ℓ = argmin J(y; φ_ℓ)：抽象层级 ℓ 下的最优网络设计
- J(·; φ_Full)：用 Full 模型评估目标函数

**Δ_ℓ ≥ 0 必须成立**，因为 y* 是 Full 口径下的全局最优，任何其他方案的 Full 评估值不可能更低。

## 四、想定设计

### 4.1 小规模存在性证明（E0, E1, E0b）

**E0（证 P1）**：2终端、3走廊。T1 只允许 {N, B} 口，T2 只允许 {S, B} 口。A0 看不见接口 → 选最便宜的 N → T2 端不可行 → 惩罚。

**E1（证 P2）**：2终端、3走廊。T2 的 wide_ring 空域脚印阻塞中心走廊 C。A1 看不见脚印 → 选 C → 被阻塞 → 惩罚。

**E0b（渐变 regret）**：源→目的、2走廊接2个不同接口。H1 接 direct 口（延迟低, cost=1.02），H2 接 detour 口（延迟高, cost=1.00）。A0 只看聚合延迟被 H2 便宜走廊吸引，A1 看接口级延迟发现 H1 总成本更低。→ 证明 regret 不仅是二值开关，也有连续值。

### 4.2 城市级参数扫描

- 8 个终端，随机放置 20×20 nm²
- 距离 < 15 nm 的终端对生成候选走廊（约 20-27 条）
- 40% OD 对有随机需求
- 每终端 1 种样式（外生固定），每走廊对应一个接口方向（direct 或 detour）
- 参数网格：

| 需求水平 | 每 OD 基础需求 |
|----------|---------------|
| low | 3 |
| medium | 8 |
| high | 15 |

| 脚印约束 | 终端有阻塞的比例 |
|----------|-----------------|
| none | 0% 终端有脚印阻塞 |
| moderate | 30% 终端阻塞 1 条走廊 |
| severe | 60% 终端阻塞 1-2 条走廊 |

- 每组合 5 个随机种子 → 共 3×3×5=45 轮
- 每轮：Full 求 y* → A0/A1/A2 各求 ŷ_ℓ → Full 评估 → 计算 Δ_ℓ
- 5 种子平均后填入 3×3 的 regime map

## 五、算法机制

### 5.1 上层网络设计问题：两阶段随机 MIP

**目标函数**：
```
min J = Σ_e act_e · x_e                              (走廊激活固定成本)
      + Σ_ω p_ω [ Σ_e c_e · f_e^ω                   (走廊流量可变成本)
                 + Σ_t ψ_t^ω                          (终端广义成本 Ψ_t)
                 + κ · unmet^ω ]                       (未满足需求惩罚)
```

**决策变量**：
- x(e) ∈ {0,1}: 走廊是否激活
- z(t,k) ∈ {0,1}: 终端样式选择（当前实验中外生固定，每终端仅 1 种样式）
- f(e,ω) ≥ 0: 走廊流量
- ψ(t,ω) ≥ 0: 终端广义成本

**约束**：
- C1: f_e ≤ cap_e · x_e（走廊容量）
- C2: x_e ≤ Σ_k feasCoeff(t,k,e) · z(t,k)（走廊可行性——**这里是抽象层级的关键差异点**）
- C3: Σ_k z(t,k) = 1（每终端选一种样式）
- C4: Σ_e f_e + unmet ≥ demand（需求满足）
- C5: λ_{t,h} = Σ_{e∈接口h} f_e（接口负荷 = 对应走廊流量之和）
- C6a: Σ_h λ_{t,h} ≤ μ_t（终端总容量）
- C6b: λ_{t,h} ≤ capMax_h（接口有效域上限）
- C7: ψ_t = 分段线性化(λ)（Ψ_t 嵌入目标函数）

### 5.2 抽象层级如何影响 MIP

不同层级用不同的 **plugin**（插件），plugin 决定了 MIP 约束的松紧：

| 方法 | A0 | A1 | A2/Full |
|------|----|----|---------|
| `isCorridorFeasible` | 恒 true | 查接口矩阵 | 查接口矩阵 |
| `isCorridorBlocked` | 恒 false | 恒 false | 查脚印冲突 |
| `getPsiBreakpoints` | **标量总负荷一元模型** | **接口分解多元模型** | **接口分解+外部性** |

**C2 约束差异**：A0 的 feasCoeff 恒为 1 → 任何走廊都能激活。Full 的 feasCoeff 会因接口不兼容或脚印阻塞而为 0 → 某些走廊被禁止。

**C7（Ψ_t）差异**：
- A0：总负荷 λ_total = Σf_e，用 D_agg(λ_total) 做一元分段线性化。8 个断点，增量 delta + 二值排序变量
- A1/Full：每接口 λ_h = Σ_{e∈h} f_e，各接口独立分段线性化。保留接口级延迟差异

### 5.3 终端广义成本 Ψ_t

```
Ψ_t = η · L̃_t + ξ · X̃_t
```
- L̃_t = L_t / L^ref（无量纲化延迟负荷）
- X̃_t = X_t / X^ref（无量纲化外部性）
- L_t = Σ_h λ_{t,h} · D_{t,h}(λ_{t,h})，D 是 M/M/c 排队延迟（非线性凸函数）
- X_t = X̄_t + Σ_h χ_h · λ_{t,h}（基础外部性 + 边际外部性）
- 当前实验：η=1, ξ=0.1

### 5.4 Evaluator（真值评估器）

**不用分段线性近似**，而是用精确的 D(λ) 函数重新计算目标：
1. 走廊激活成本：直接累加
2. 流量成本：检查两端可行性——不可行的走廊，其流量按 unmet 惩罚计
3. 终端成本 Ψ_t：用接口级负荷精确计算 L_t = Σ_h λ_h · D_h(λ_h)
4. 外部性：ξ>0 时通过 Ψ_t 公式计入

### 5.5 Regret 计算流程

1. 预计算所有 (终端, 样式) 的 TerminalResponse（容量、延迟函数、接口、脚印等）
2. Full plugin + TwoStageMIP.solve() → y*, J_MIP*
3. Evaluator(y*, Full) → J*（精确评估的真值最优）
4. 对每个层级 ℓ：plugin_ℓ + TwoStageMIP.solve() → ŷ_ℓ
5. Evaluator(ŷ_ℓ, Full) → J(ŷ_ℓ; Full)
6. Δ_ℓ = J(ŷ_ℓ; Full) - J*

| 实验 | A0 Δ | A1 Δ | A2 Δ | 结论 |
|------|------|------|------|------|
| E0 | +487.5 | 0.0 | 0.0 | P1 ✅ |
| E1 | +490.0 | +490.0 | 0.0 | P2 ✅ |

### 6.2 城市级 Regime Map（5 种子平均）

**最小充分层级（|相对 regret| < 1% 判定）**：
```
              none        moderate    severe
low           A0          A0          A2
medium        A0          A2          A2
high          A0          A0          A2
```

**相对 Regret A0 (%)**：
```
              none        moderate    severe
low           -31.5       -37.5       620.5

```

**相对 Regret A1 (%)**：
```
              none        moderate    severe
low           0.0         96.1        671.7
medium        0.0         72.2        417.1
high          0.0         23.0        19.9
```

**A2 在全部 45 轮中 regret 恒为 0。**

### 6.3 结果异常

**🔴 A0 负 regret 问题**

A0 在 footprint=none 下全部负 regret（-31% 到 -51%），在 moderate 下也有负值。这意味着 J(ŷ_A0; Full) < J(y*; Full)，即 A0 的方案在 Full 评估下"优于" Full 自己的最优解。

**这在数学上不可能**，因为 y* 是 Full 口径下的全局最优。必有 bug。

## 七、对负 regret 的初步猜测

1. **MIP 分段线性化 vs Evaluator 精确计算不一致**：
   - TwoStageMIP 用 8 段 PwL 近似 Ψ_t → 嵌入 MIP 约束是 ψ_t = PwL(λ)（等式约束）
   - Evaluator 用精确 D(λ) 重新算 → 数值不同
   - 如果 PwL 在某些区间高估了 Ψ_t，Full MIP 的最优解 y* 会避开那些"看起来贵"的方案，但 Evaluator 用精确值发现其实没那么贵 → y* 在 Evaluator 口径下可能不是最优

2. **A0 和 Full 的可行域形状不同**：
   - Full 按接口分解 → 每接口独立 PwL → 联合可行域更受限
   - A0 按总负荷一元 PwL → 可行域更宽松
   - Full MIP 在更紧的可行域里求最优 → y* 可能是 Evaluator 口径下的一个次优解

3. **MIP solver gap tolerance**：intlinprog 默认可能不够紧
