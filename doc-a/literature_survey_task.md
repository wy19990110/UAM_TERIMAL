# 文献调研任务书

## 调研目的

为论文 **"面向低空可复用航路网络设计的终端区决策保持抽象"** 寻找 baseline 模型。

我们的核心研究问题是：**终端区（vertiport）需要被抽象到什么粒度，才能保持上层走廊网络设计决策不变？**

为此，我们需要找到一篇（或几篇）做 **UAM/低空走廊网络设计** 的论文，它具有完整的数学优化模型（目标函数+约束+决策变量），且对终端做了某种简化假设。我们将以它的模型为起点，升级终端建模，证明简化导致了错误的网络设计决策。

## 已排除的方向

以下类型的论文**不是**我们要找的 baseline：

1. **Vertiport 选址/设施选点**（如 Wu & Zhang 2021, He et al. 2026）——决策是"选哪些 vertiport"，不是"选哪些走廊"
2. **纯飞行路径规划/冲突消解**（如 Tang & Zhang 2021）——关注单机 4D 轨迹，不是网络级走廊选择
3. **交通分配/模式选择**——只做需求分配，不做网络拓扑设计
4. **综述/定性分析**——没有数学模型

## 要找的论文特征

**必须满足全部条件**：

| 特征 | 说明 |
|------|------|
| 决策层 | 给定（或同时决定）vertiport 位置后，**决定哪些走廊/航路激活、流量如何分配** |
| 数学模型 | 有明确的**目标函数、决策变量、约束条件**（MIP、LP、或其他优化形式） |
| 走廊/弧/航路 | 模型中有**候选走廊集合**，需要决定哪些开通（不是完全图直连） |
| 终端/节点 | 终端在模型中至少有**容量**属性（哪怕只是一个容量上界），理想情况有延迟/成本函数 |
| 应用领域 | UAM / AAM / 低空物流 / eVTOL / 无人机物流网络（不是传统航空） |

**加分条件**（非必须但优先）：

- 终端有多个接口/进离场方向（approach fix）
- 模型中有流守恒约束（多商品流）
- 考虑了终端空域占用/脚印
- 有容量化延迟函数（不仅是硬容量上界）
- 问题规模是城市级（8-50 个 vertiport）

## 背景知识补充

### 我们论文的抽象层级体系

- **A0**：终端 = (μ, D) — 总容量 + 聚合延迟函数。终端是黑箱。
- **A1**：终端 = (A, μ, D) — + 接口可行矩阵 + 接口级延迟。知道哪条走廊能接哪个进近口，各口延迟不同。
- **A2**：终端 = (A, μ, D, V, X) — + 空域脚印 + 外部性。终端场型的空域占用会阻塞附近走廊。
- **Full**：完全信息。

### 我们要证明的

1. 存在网络实例使 A0 的最优走廊设计在真值评估下产生正 regret（A0 不充分）
2. 存在网络实例使 A1 的最优走廊设计在真值评估下产生正 regret（A1 不充分）
3. 在不同（需求×脚印约束）组合下，最小充分抽象层级不同（regime map）

### 典型的走廊网络设计模型长什么样（供参考）

经典的 Fixed-Charge Multicommodity Network Design (MCND)：
```
min  Σ_e f_e × x_e  +  Σ_e,k c_e × y_e^k
s.t. Σ_{e∈δ+(v)} y_e^k - Σ_{e∈δ-(v)} y_e^k = b_v^k    ∀v,k  (流守恒)
     Σ_k y_e^k ≤ u_e × x_e                                 ∀e    (容量+激活)
     x_e ∈ {0,1}, y_e^k ≥ 0
```
其中 x_e 是弧激活、y_e^k 是商品 k 在弧 e 上的流量、f_e 是固定成本、c_e 是可变成本、u_e 是容量、b_v^k 是节点 v 对商品 k 的供需。

如果有论文在 UAM 场景下用了类似框架，且节点（vertiport）有容量约束，那就是理想的 baseline。
