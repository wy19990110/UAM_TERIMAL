# 实验说明：验证目标、想定设计、算法机制、实验结果

## 一、要验证的结论

**核心研究问题**：在城市空中交通(UAM)网络设计中，对终端区域(vertiport)做不同程度的抽象建模，会导致上层网络设计决策产生多大偏差？

具体地，我们定义了 4 个抽象层级（信息可见性从低到高）：

| 层级 | 可见信息 | 直觉含义 |
|------|----------|----------|
| **A0** | (μ, D) — 总容量 + 聚合延迟 | 把终端当"黑箱"，只告诉你吞吐量和平均延迟 |
| **A1** | (A, μ, D) — 接口可行矩阵 + 容量 + 接口级延迟 | 知道哪些走廊能接哪些进近口，各口的延迟函数不同 |
| **A2** | (A, μ, D, V, X) — + 空域脚印 + 外部性 | 知道终端场型的空域占用会阻塞附近走廊、噪声影响 |
| **Full** | 完全信息（真值基准） | 接口+脚印+外部性+资质规则全部可见 |

**要证明的三条主张**：

1. **P1**：存在网络实例使 A0 的最优决策在真值评估下产生正 regret（A0 不充分）
2. **P2**：存在网络实例使 A1 的最优决策在真值评估下产生正 regret（A1 不充分）
3. **P3（城市级）**：在不同的需求×脚印约束组合下，最小充分抽象层级不同——形成"regime map"

**Regret 定义**：
```
Δ_ℓ = J(ŷ_ℓ; Full) - J(y*; Full)
```
- ŷ_ℓ = 在抽象层级 ℓ 下求解得到的网络设计方案
- y* = 在 Full（完全信息）下求解得到的真值最优方案
- J(·; Full) = 用 Full 模型评估方案的目标函数值
- Δ_ℓ ≥ 0 表示抽象不足导致的决策损失

## 二、想定设计

### 2.1 小规模存在性证明

**E0（证明 A0 不充分）**：
- 2 终端 T1, T2；3 走廊 N(cost=1), S(cost=4), B(cost=2.5)
- T1 只允许走廊 {N, B}，T2 只允许 {S, B}（接口可行矩阵）
- A0 看不见接口限制 → 选最便宜的 N → 但 T2 端不可行 → 惩罚
- A1/Full 看见限制 → 选 B（唯一两端都行的）→ J*=12.5

**E1（证明 A1 不充分）**：
- 2 终端 T1, T2；3 走廊 C(cost=1), N(cost=2), S(cost=2)
- T2 的 wide_ring 样式的空域脚印阻塞走廊 C
- A1 看不见脚印 → 选 C → 被阻塞 → 惩罚
- A2/Full 看见脚印 → 避开 C，选 N+S

**E0b（渐变 regret，非二值）**：
- 源 S → 目的 T，2 走廊接 T 的两个不同接口
- H1 接 direct 进近（延迟低, cost=1.02），H2 接 detour 进近（延迟高, cost=1.00）
- A0 只看聚合延迟 → 被 H2 更低的走廊成本吸引
- A1 看接口级延迟 → 发现 H1 总成本（走廊+终端延迟）更低

### 2.2 城市级参数扫描

- **8 个终端**，随机放置在 20×20 nm² 区域
- 距离 < 15 nm 的终端对生成走廊（约 20-27 条）
- 40% 的 OD 对有随机需求
- **参数网格**: 3 需求水平 × 3 脚印水平 × 5 随机种子 = 45 轮

|------|-----|--------|------|

|------|------|----------|--------|
| 终端脚印阻塞比例 | 0% | 30% (阻塞1条) | 60% (阻塞1-2条) |

## 三、算法机制

### 3.1 TwoStageMIP（核心优化器）

**目标函数**：
```
minimize J = Σ_e act_e · x_e                     (走廊激活固定成本)
           + Σ_ω p_ω [ Σ_e c_e · f_e^ω           (走廊流量可变成本)
                      + Σ_t ψ_t^ω                  (终端广义成本)
                      + κ · unmet^ω ]               (未满足需求惩罚)
```

**决策变量**：
- x(e) ∈ {0,1}: 走廊是否激活
- z(t,k) ∈ {0,1}: 终端样式选择（当前仅单样式，z 固定为 1）
- f(e,ω) ≥ 0: 走廊流量
- ψ(t,ω) ≥ 0: 终端广义成本

**关键约束**：
- C1: f ≤ cap·x（走廊容量上界）
- C2: x ≤ Σ feasCoeff·z（走廊可行性，依赖 plugin 的 isCorridorFeasible + isCorridorBlocked）
- C3: Σz = 1（样式唯一）
- C4: Σf + unmet ≥ demand
- C5: λ_{t,h} = Σ_{e∈接口h} f_e（负荷-流量链接）
- C6a: Σ_h λ_{t,h} ≤ μ_t（总终端容量）
- C6b: λ_{t,h} ≤ capMax_h（接口有效域）
- C7: ψ ≥ 分段线性化(λ)（Ψ_t 的 SOS2 近似）

**抽象层级的差异体现在 plugin 的返回值**：
- A0 的 `isCorridorFeasible` 恒返回 true → C2 对 A0 无效 → A0 可以激活任意走廊
- A0 的 `isCorridorBlocked` 恒返回 false → C2 不阻塞
- A0 的 `getPsiBreakpoints` 返回 `isPerInterface=false` → C7 用**总负荷一元模型**
- A1 的 `isCorridorFeasible` 查询接口矩阵 → C2 约束走廊必须匹配接口
- A1 的 `isCorridorBlocked` 恒返回 false → 无脚印阻塞
- A1 的 `getPsiBreakpoints` 返回 `isPerInterface=true` → C7 按**接口分解**
- Full 的所有方法都查完整 TerminalResponse → C2 同时约束可行性和脚印阻塞

### 3.2 Ψ_t 分段线性化

终端广义成本 Ψ_t = η · L̃_t + ξ · X̃_t

其中 L̃_t = L_t / L^ref（无量纲化延迟）, X̃_t = X_t / X^ref（无量纲化外部性）

L_t = Σ_h λ_{t,h} · D_{t,h}(λ_{t,h})，其中 D 是 M/M/c 排队延迟函数（非线性凸函数）。

为嵌入 MIP，使用分段线性化：
- 对每个接口 h 的 L_{t,h}(λ) = λ · D(λ) 曲线取 8 个均匀断点
- 用递增 delta 变量 + 二值排序变量实现 SOS2 式逼近
- 约束: ψ_t = Σ_h (分段线性(λ_h))

**关键区别**：
- A0 对**总负荷**做一元分段线性化 → 聚合后的 D_agg 无法区分接口
- A1/Full 对**每接口负荷**分别分段线性化 → 保留接口级延迟差异

### 3.3 Evaluator（真值评估器）

对给定设计方案 + 给定 plugin，重新计算目标函数值：
1. 走廊激活成本：直接累加
2. 流量成本：检查每条走廊两端可行性——**如果不可行，流量按 unmet 惩罚计算**
3. 终端成本 Ψ_t：用接口级负荷重新计算 L_t（真值模型，非分段线性近似）
4. 外部性成本：ξ>0 时计入，ξ=0 时用旧的外部性公式

### 3.4 RegretFramework（编排器）

流程：
1. 预计算所有 (终端, 样式) 对的 TerminalResponse
2. 构建 Full plugin → TwoStageMIP.solve() → 得到真值最优 y*, J*
3. 对每个抽象层级 ℓ：构建 plugin_ℓ → TwoStageMIP.solve() → 得到 ŷ_ℓ
4. 用 Evaluator 在 Full plugin 下评估 ŷ_ℓ → 得到 J(ŷ_ℓ; Full)
5. Δ_ℓ = J(ŷ_ℓ; Full) - J*

## 四、实验结果

### 4.1 小规模实验（均通过）

| 实验 | A0 regret | A1 regret | A2 regret | 结论 |
|------|-----------|-----------|-----------|------|
| E0 | +487.5（惩罚） | 0.0 | 0.0 | A0 不充分 ✅ |
| E1 | +490.0（惩罚） | +490.0（惩罚） | 0.0 | A1 不充分 ✅ |
| E0b | +2.34（连续值） | 0.0 | 0.0 | 渐变 regret 存在 ✅ |

### 4.2 城市级参数扫描（45 轮完成）

**Regime Map（5 种子平均后的最小充分层级）**：

```
              none        moderate    severe
low           A0          A0          A2
medium        A0          A2          A2
high          A0          A0          A2
```

**相对 Regret A0 (%) — J(ŷ_A0; Full)/J* - 1**：
```
              none        moderate    severe
low           -31.5       -37.5       620.5
medium        -38.9       67.7        453.4
high          -50.6       -3.4        26.2
```

**相对 Regret A1 (%)**：
```
              none        moderate    severe
low           0.0         96.1        671.7
medium        0.0         72.2        417.1
high          0.0         23.0        19.9
```

**A2 在所有 45 轮中 regret 恒为 0。**

### 4.3 结果中的异常

**🔴 A0 大面积负 regret（最高优先级问题）**

在 footprint=none 和部分 moderate 条件下，A0 的 regret 是负数（-3.4% 到 -50.6%）。这意味着 A0 选出的方案在 Full 评估下**优于**真值最优 y*。

理论上 Δ_ℓ = J(ŷ_A0; Full) - J(y*; Full) ≥ 0 必须成立，因为 y* 是 Full 模型下的全局最优。

可能原因猜测：
1. **Evaluator 和 TwoStageMIP 的 Ψ_t 计算不一致**：TwoStageMIP 用分段线性近似，Evaluator 用精确 D(λ) 值；如果 PwL 是上界近似（凹壳），MIP 会高估 Ψ_t → 真值最优 y* 可能不是 Evaluator 口径下的最优
2. **A0 和 Full 的 Ψ_t 线性化方式不同**：A0 用标量模式、Full 用接口分解模式 → 两者的可行域形状不同 → Full 的 MIP 约束更紧
3. **MIP 求解器未达全局最优**：intlinprog 默认 gap tolerance 可能太松

**🟡 A1 在 footprint=none 下 regret 恒为 0**

A1 和 Full 在无脚印条件下可能共享完全相同的约束集（因为 Full 的额外信息只是脚印阻塞和外部性，而 none 条件下无阻塞、ξ=0.1 的外部性影响可能太小）。

需要确认：A1 和 Full 的 getPsiBreakpoints 在 ξ=0.1 时的输出差异。
