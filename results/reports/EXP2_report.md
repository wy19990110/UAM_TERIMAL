## 2. 实验目的与预期

### 2.1 本次实验目的
证明即使 admissibility 一样（两条边都可行），只用 aggregate service（M0）也会误导网络设计，因为 port 级 service 不对称改变了最优连边和分流。

### 2.2 对应假设/预期现象
- 低负荷时 M0 与 M1 接近（二次项占比小）
- 中高负荷时 M0 regret 随 ρ 和 κ_S 上升
- M1（per-port service）基本恢复 benchmark

### 2.3 与实验链的关系
- 上游依赖：EXP-1 Go（已满足）
- 下游影响：Go → 可进入 EXP-3

### 2.4 成功判据（预注册）
在某个参数区间内，M0 regret 随 ρ 或 κ_S 呈单调或近单调上升。

## 9. 初步分析

### 9.1 结果是否符合预期
**完全符合预期。**
- ρ=0.4（低负荷）：服务成本二次项 b·λ² 很小，travel cost 主导 → M0 不犯错 ✓
- ρ≥0.6：二次项显著，M0 看不见 port 差异 → 把流量送到贵 port → regret 上升 ✓
- M1 regret 恒为 0 ✓

### 9.2 对研究问题的含义
**S 信息不可只做 aggregate。** 当需求强度中等以上（ρ≥0.6）时，port 级 service 不对称导致 M0 的决策系统性偏向"外部便宜但终端贵"的方案。M1（per-port service）完全纠正。

### 9.3 错误归因分析
M0 的 regret **100% 来自 S 信息聚合损失**（A 在此实验中两边都可行，不贡献 regret）。

具体机制：M0 用 aggregate b̄ 估计服务成本，无法区分 flow 走 h1(b=0.3) 还是 h2(b=0.3κ_S)。当 travel cost 引导 flow 走向 h2（贵 port），M0 低估了该方案的终端成本。

### 9.4 Regime 边界与阈值效应

**ρ 的阈值效应**：
- ρ=0.4 → ρ=0.6 之间存在**锐利跳变**：从 0% regret 直接跳到 3.9%~8.4%
- 这是因为低负荷时 b·λ² ≈ 0，一旦 λ 达到某个值，二次项开始主导

**κ_S 的单调递减效应**：
- 反直觉但正确：κ_S 越大，M0 regret 越小
- 原因：κ_S 极大时，M* 也避开贵 port（只选单边）→ 与 M0 一致
- **最大 regret 出现在 κ_S=1.0（对称）**——此时 M* 想分流到两边，但 M0 只选一边

**Δc 几乎无影响**（7.5%~7.8%）：travel cost gap 不是主要驱动因素。

### 9.5 异常迹象
无。所有 regret ≥ 0，M1 regret 全为 0。

### 9.6 Stop/Go 判定
**Go。** M0 regret 随 ρ 从 0% 单调上升到 23.6%，满足"在某个参数区间内明显上升"的 Go 条件。
