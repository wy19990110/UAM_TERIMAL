## 2. 实验目的与预期

### 2.1 本次实验目的
量化 "JO 很贵"——随 nT/nW 增大，记录各方法的 runtime + gap to JO，画 quality-time 前沿图。

### 2.2 对应假设/预期现象
- B0/B1：便宜但质量差
- JO：质量高但代价大

- PR quality-time 位于 B0 和 JO 之间

## 5. 分析

### 5.1 成功判据检验
- ✅ JO runtime 超线性增长：S3(0.2s) → S16(27.6s)，增长 **138x**
- ⚠ PR quality-time 在 S8 处优异，但 S12/S16 有离群值

### 5.2 关键发现

1. **S8 是甜点**：M2/PR gap 仅 4.3%，接近 JO 质量。JO time=0.3s, PR time=0.3s。此规模下分层方法几乎无损。

2. **S12/S16 离群值**：
   - S12 seed=1: M2 gap=445%，而 seed=2/3 接近 0%
   - S16 seed=1: M2 gap=528%，而 seed=2/3 接近 0%
   - 这说明大规模下 solver 对图结构敏感——某些 seed 生成的图使 MILP 找到差的解。

3. **JO/PR runtime ratio 仅 2.3x**：远小于预期（应为 10-100x）。原因可能是当前 S16 (nT=16) 仍不够大，或 intlinprog 在两种问题上的 scaling 行为相近。

4. **PR ≡ M2**：因 rule 门槛过低，PR 总是选 M2。当 M2 结果好时 PR 好，M2 差时 PR 一样差。

5. **B0 在 S8 上 gap 极大（10000%+）**：说明 8+ terminals 时 incumbent 方法完全不可用。

### 5.3 改进建议
- 增加种子数到 10（特别是 S12/S16），减少离群值影响
- 增加更大规模 S20/S24 以展示 runtime ratio 的分离
- 考虑对 MILP solver 设置 time limit，观察 gap 随时间的收敛

---

## 6. 结论

- JO 运行时间从 S3 到 S16 增长 138x，超线性 scaling 得到验证
- S8 是 quality-time 的甜点（gap=4.3%, time=0.3s）
- S12/S16 的离群值和 PR≡M2 问题需要在后续工作中解决
- 论文中应用 median 而非 mean 展示 S12/S16 数据（排除离群值影响）
